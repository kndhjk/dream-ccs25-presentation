👤 Speaker 1 — Part 1
Part 1 · How People Try to Fix It ~3 min
The Standard Two-Step Fix
Most research follows this pattern.
Step 1

DRIFT DETECTION — Find the weird samples

Look at new incoming samples. Decide: has this malware changed enough to fool our model? Pick the suspicious ones.

Who looks weird? → pick those
Step 2

DRIFT ADAPTATION — Update the model

Send weird samples to malware experts. Experts label them. Add labeled samples to training set. Retrain the model.

New labeled data → retrain

Problem 1: Detection is blind to the classifier

Most drift detectors train their OWN model to decide what's weird. They ignore what the target classifier actually uses. It's like a doctor diagnosing a patient without knowing what the patient already knows.

Problem 2: Experts waste their knowledge

When experts analyze malware, they do deep static and dynamic analysis. They understand the WHY. But the model only gets a simple LABEL. All the reasoning is thrown away. We need way too many labeled samples to make updates work.

👤 Speaker 2 — Part 2
Part 2 · Concept Learning ~3 min
How Dream Learns Malware Concepts

What Is a 'Concept' Here?

A concept = a TYPE of malicious behavior. Like:

  • Stealing private info — SMS, contacts, photos
  • Remote control by the attacker
  • Silent downloads without asking
  • Stealing bank or financial info
  • Ransomware — encrypt your files
  • Abusing premium SMS services

Dream puts each sample into a LATENT SPACE where each direction is one of these behaviors.

Two Signals at the Same Time

Signal 1

Supervised Learning

Link latent directions to the classifier's internal activations. We want to know: what is the classifier actually FEARING when it sees this behavior?

Signal 2

Contrastive Learning

Samples with the SAME concept stay close in the latent space. Samples with DIFFERENT concepts stay apart. Good separation.

L_total = λ₀·L_rec + λ₁·L_sep + λ₂·L_pre + λ₃·L_rel
reconstruction loss + contrastive loss + concept presence loss + concept reliability loss
👤 Speaker 2 — Part 2
Part 2 · Detection Trick ~3 min
The Drift Detection Trick
How to find drift WITHOUT touching any training data.
① New sample x arrives
→
② Detector rebuilds x → x̂
→
③ Feed BOTH to classifier
→
④ Compare predictions
→
⑤ Drift score!

Predictions agree ✓

Good. The concepts used to understand this sample are still reliable. The classifier still "gets it." Low drift score. Everything is fine.

Predictions disagree ✗

Bad. The concepts used to rebuild this sample don't match what the classifier expects. The classifier's understanding is OUTDATED. High drift score. → Alert!

Why This Trick Is So Good

The detector uses THE SAME classifier it is monitoring. No other reference data needed. If the classifier changes its mind after seeing the rebuilt version — something genuinely changed in the malware world. That's a real drift.

👤 Speaker 2 — Part 2
Part 2 · Adaptation Method ~3 min
Making Adaptation Actually Work

Old Way — Label Only

  • Expert studies a malware sample carefully
  • Expert says: this is family X
  • Model only gets the label. All the rich analysis — gone.

Result: need TONS of samples

To get a model update to work, you might need 80, 100, even more labeled samples. Expensive and slow.

Dream's Way — Label PLUS Concept

  • Dream shows WHICH concept caused the drift
  • Expert says: label X AND confirms/changes the concept explanation
  • Both classifier AND detector update together — a joint update

Result: far fewer samples needed

The expert's reasoning is now PART of the update. The model understands not just WHAT changed, but WHY. Much more efficient.

👤 Speaker 3 — Part 3
Part 3 · Experimental Setup ~2 min
How We Tested Dream

Two Datasets

  • Drebin — 3,317 samples, 8 malware families, years 2010–2012
  • Malradar — 2,589 samples, 8 families, years 2015–2021
  • Malradar has 10 behavioral concepts labeled for each sample
  • Extended test: 4,410 samples, 180 families, 2015–2020

Three Classifiers

  • Drebin classifier — binary feature vectors, simple MLP with two hidden layers (100 and 30 neurons)
  • Mamadroid classifier — API call pairs abstracted to package call pairs, Markov chain + MLP (1000 and 200 hidden)
  • Damd classifier — raw opcode sequences, CNN with two conv layers (64 filters each), vocabulary size 218

All are real, widely-used Android malware classifiers.

Testing Method
Hold-out by family: exclude each family from training, use it ONLY for testing — simulates a brand new family appearing
Time-based separation followed. Creates 8 classifiers per dataset (one per held-out family).
