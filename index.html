👤 Speaker 1 — Part 1
Part 1 · Background
What Is Concept Drift?
When malware changes, classifiers forget.
🦠

Malware Changes Fast

Every month, over 3 million new malware samples are found. Attackers always find new ways to hide.

📉

Classifier Gets Worse

A model trained on old data can drop to almost random — like guessing. This is concept drift.

🔄

Two Types of Drift

Intra-class: new version of the same family. Inter-class: a completely new family shows up.

Real Example
Xavier malware — 5 versions in 8 months
Intra-class drift: same family, new versions keep appearing

The Core Problem

We have a classifier. It was good before. But the malware world changed. Now the classifier keeps making wrong calls — and we need a way to know WHEN it is going wrong and HOW to fix it.

👤 Speaker 1 — Part 1
Part 1 · Why Old Ways Fail
Why Old Ways Don't Work

Detection Problem

  • Old detectors learn their OWN idea of what malware looks like — they ignore what the target classifier actually uses.
  • They also need to look at training data all the time — not safe for real deployment.

Adaptation Problem

  • When experts label samples, they do deep analysis — static analysis, dynamic analysis, rich reasoning.
  • But the model only gets a simple LABEL. All that rich thinking is thrown away.
  • So we need to label way too many samples to make the model update work.

Result

High cost. Slow response. Model still gets things wrong.

The Standard Two-Stage Plan
Every good drift solution has two steps.
🔍

Step 1 — Detect

Look at new samples. Decide: has this malware changed enough to fool our model? Pick the ones that look weird.

Drift Detection
🔄

Step 2 — Adapt

Send weird samples to experts. Experts label them. Add labeled samples to training set. Retrain the model.

Drift Adaptation
💡

Dream's Twist

We make both steps better. Detection is smarter. Adaptation uses expert knowledge fully — not just a label.

Active Learning Loop

The Gap in Existing Work

Most papers focus on detection. Very few try to make the ADAPTATION step better. Dream does BOTH.

👤 Speaker 2 — Part 2
Part 2 · Concept Space
How Dream Learns Concepts

What Are Concepts Here?

A concept = a type of malicious behavior. Like:

  • Stealing privacy info (SMS, contacts...)
  • Remote control by attacker
  • Silent downloads
  • Bank info stealing
  • Ransomware
  • Premium service abuse

Dream maps each sample into a latent SPACE where each direction means one of these behaviors.

How Does Dream Learn Them?

Two signals at the same time:

  • Supervised: link latent directions to classifier's internal activations
  • Contrastive: samples with same concept stay close; different concepts stay apart
L = λ₀L_rec + λ₁L_sep + λ₂L_pre + λ₃L_rel

The detector learns what the CLASSIFIER cares about — not something independent.

👤 Speaker 2 — Part 2
Part 2 · Detection Method
The Drift Detection Trick
A clever way to find drift without touching training data.
① New sample x comes in
→
② Detector rebuilds x → x̂
→
③ Feed x AND x̂ to classifier
→
④ Compare the two predictions
→
⑤ Drift Score!

Predictions Agree?

Good. The concepts used to understand x are still reliable. Low drift score.

Predictions Disagree?

Bad. The concepts that built x̂ don't match what the classifier expects. High drift score. → Alert!

Why This Is Smart

The detector uses the SAME classifier it is monitoring. No other data needed. If the classifier changes its mind after seeing a rebuilt version — something is wrong.

👤 Speaker 2 — Part 2
Part 2 · Adaptation Method
How Experts Help Fix the Model

Old Way

Expert looks at a sample.

Expert says: this is family X.

Model only gets the label. All the WHY is lost.

Problem

Need lots and lots of labeled samples to update the model.

Dream's Way

Expert looks at a sample.

Dream shows: which CONCEPT caused the drift? (e.g., unusual SMS-stealing behavior)

Expert says: label AND concept revision.

Better

Model gets rich guidance. Updates are much more efficient. Far fewer samples needed.

The Joint Update
Classifier + Detector both update together
Expert feedback flows back into BOTH models — making future detection more aligned
